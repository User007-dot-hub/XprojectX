# Plan 4.1: Retraining UI Integration

## Objective
Provide the user with a distinct mechanism to trigger a manual model retraining flow directly from the dashboard, ensuring they visually understand training is in progress.

## Context
- .gsd/SPEC.md
- frontend/src/App.jsx
- frontend/src/components/PredictionResult.jsx

## Tasks

<task type="auto">
  <name>Implement Retrain Button and Logic</name>
  <files>frontend/src/App.jsx, frontend/src/components/PredictionResult.jsx</files>
  <action>
    - Add a `handleRetrain` async function structurally parallel to `handlePredict` within `App.jsx` that executes POST to `/api/retrain` with `{ticker, timeframe}`.
    - Set a semantic `isRetraining` loading state tracking specifically the retraining delay which spins off an internal component loader (e.g. "Retraining model...").
    - Call the backend correctly. After a 200 OK successful train, immediately invoke `handlePredict` internally to update prediction numbers under the hood.
    - Map `handleRetrain` binding to a new "Retrain" button situated within `PredictionResult.jsx` next to the baseline confidence stats. Use the `RefreshCw` icon from lucide-react. Style identically mapping to the glass UI schema.
  </action>
  <verify>Check that `App.jsx` securely executes and UI resolves without crashing.</verify>
  <done>Retrain triggers backend ML pipeline training mechanisms sequentially.</done>
</task>

## Success Criteria
- [ ] User can click Retrain; UI logs explicit progress blocks while processing.
- [ ] Dashboard repopulates immediately post-train incorporating the freshly learned numbers.
